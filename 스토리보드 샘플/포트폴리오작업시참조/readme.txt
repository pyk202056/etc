팀 별 프로젝트 작성 및 디버깅

1. 팀 별 프로젝트 구현, 업무분장에 따른 프로젝트 진행상황 점검 및 토의.
   보고서 제출 내용
( Project Outline, Project Overview, Site Menu 구조도, 구현 기술, 업무 분장표, 
Project 일정, Site Layout, Design Strategy, Web Profile menu )
    
기술 예)
- 프로젝트명: 사이트명 또는 사이트 url을 기입.
- 작성자 : 해당 페이지 스토리보드를 작성한 작성자의 소속부서와 성명을 기입.
- Page 번호 : 스토리 보드의 분실을 막고, 화면전환 표시로 사용.
- 화면 ID : 이 화면을 저장할 파일 이름을 기입.
*             예) about.txt 
- 화면주제 : 본 화면의 내용, 용도를 쉽게 알 수 있도록 이름 짓는다
*             예) 회사소개, 
- 화면경로 : 홈 화면에서 부터의 현재 위치를 나타내는 경로를 기입.
*             예) 홈/회사소개/회사연혁 
- 화면구성 : 각 작업자가 화면의 내용과 구성을 시각적으로 쉽게 이해 하도록 스케치로 표현. 
- 화면설명 : 화면구성에서 위치를 선정해 놓은 구성요소들에 대해 구체적으로 표기. 
- F/C와 Logic : 폴더경로를 자세하게 표시. 
이것이 명확히 표기되어야만 다음 관리자가 쉽게 업데이트 및 수정이 가능. 
- 첨부사항 기술
